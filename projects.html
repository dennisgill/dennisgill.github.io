Web-App for Simpson's Giphy (in progress)

Web-app when given a Simpson's keyword, it grabs the input from a user input box and grabs 10 static, non-animated gif images from the GIPHY API and places them on the page. If the user tries to search without providing any keyword, the app will alert the user until a keyword is provided.

Game Server for Multiplater Game

I programmed the game server using python that can receive username and password from the command line ( telnet to the server) and from within the multiplayer game. Then, it sends the JSON object with username and password fields to the web-server once the users tries to log into the game. If the users enters the valid username and password and has previously signed up at the game web-page, the web-server sends a positive acknowledgement back to the game server. If the user enters invalid info or is not signed up, then the web-server responds with an error. After a game is finished or the user quits the multiplayer game, the game server sends a JSON object to the web-server with the username and result (win) keys to notify the web server about the results of the game. Furthermore, to handle multiple games with multiple players, multithreading was employed to handle multiple connections.
